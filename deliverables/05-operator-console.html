How this console plugs into AWS

production data sources
Heatmap data
Reads aggregated Parquet from s3://incident-curated-prod/curated/_heatmap/ via Athena, surfaced through an API Gateway HTTP API + Lambda that returns GeoJSON. The Lambda role has s3:GetObject on the curated bucket only and no kms:Decrypt — the heatmap is pre-aggregated and anonymized to 3 dp, so the raw KMS-encrypted Parquet stays sealed.
Pipeline status
Step Functions ListExecutions on arn:…:stateMachine:IngestionStateMachine, paged through Lambda. Per-execution detail comes from GetExecutionHistory on click. Live status uses EventBridge → WebSocket API for push updates.
Skew monitor
AWS Glue job emits Spark metrics to the CloudWatch Glue namespace (dimensioned by JobName + JobRunId); we call cloudwatch:GetMetricData for partition-size histograms. The "after" view comes from the salted aggregator's per-bucket counts published as custom metrics via cloudwatch:PutMetricData.
Rotation status
secretsmanager:DescribeSecret on the Weather API key secret — exposes VersionIdsToStages, NextRotationDate, LastRotatedDate. We never call GetSecretValue from the console.
Audit log
CloudTrail Lake federated query for events on the KMS key ARN and the Break-Glass role ARN. Console runs read-only queries through a service Lambda with cloudtrail:StartQuery and cloudtrail:GetQueryResults. Results are cached for 60s.
Hosting & auth
Static HTML/JS served from S3 + CloudFront with OAC. Sign-in via Cognito federated to IAM Identity Center (no shared credentials). The console's API Lambda assumes a read-only role arn:aws:iam::123456789012:role/ConsoleReader — no kms:Decrypt, no secretsmanager:GetSecretValue, no s3:PutObject. Even with full UI compromise, an attacker gets the same view a stakeholder gets.
