State machine path

  1. 1S3 event starts executionInput contains bucket, key, and manifestKey for the uploaded zip.
  2. 2ValidateChecksum LambdaStreams the object, computes MD5, validates manifest format, and returns a safe copySource.
  3. 3Choice and routingFailure goes to parallel quarantine copy and SNS alert. Success goes to ExtractZip.
  4. 4Glue startJobRun.syncRuns anonymization and Parquet conversion before the execution succeeds.

Defense layers

  1. 1KMS explicit denyDenies Decrypt and ReEncryptFrom to everyone except allowed runtime principals.
  2. 2Service and MFA conditionsGlue must use kms:ViaService. BreakGlass must use recent MFA.
  3. 3Organizations SCPWorkload account admins cannot rewrite the key policy or disable audit controls.
  4. 4Audit trailCloudTrail/EventBridge detects break-glass and policy edit attempts.

Before and after

Algorithm

  1. 1Add random saltUniform 0..63 per row breaks the hot key across workers.
  2. 2Partial aggregateGroup by intersection_id and salt, emitting count and sums.
  3. 3Final aggregateDrop salt and divide sum by count, avoiding avg(avg) bias.
  4. 4Write by time/regionKeep intersection as a column to avoid write-time skew.

Rotation invariant

  1. 1createSecretNew Weather API key is issued and stored as AWSPENDING.
  2. 2setSecretNo traffic moves. The current key remains live.
  3. 3testSecretLambda calls the Weather API with AWSPENDING. Failures abort rotation.
  4. 4finishSecretOnly now does AWSCURRENT move to the new version and old key revocation begin.
