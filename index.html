01Orchestration

Automated ingestion pipeline

They asked for

A serverless S3 zip ingestion workflow using Step Functions, MD5 validation, quarantine/SNS routing, extraction, Glue anonymization, and Parquet conversion.

What I built

An ASL state machine with validation, Choice routing, parallel quarantine operations, bounded alert messages, retries with jitter, ExtractZip, and Glue startJobRun.sync.

Why it works

Poisoned files never reach analytics. Valid files are anonymized before curated storage. Operational failures alert without flooding SNS payload limits.

  • OKMD5 is streamed in Lambda and compared to a manifest field validated as 32-char hex.
  • OKFailed validation copies to quarantine with metadata and publishes an SNS alert.
  • OKPassed validation proceeds to extraction and Glue anonymization with GPS precision set to 3 decimals.

02Zero Trust

Ghost Admin defense

They asked for

A KMS key policy that lets the Glue service role decrypt but explicitly denies everyone else, including AdministratorAccess, unless a security condition is met.

What I built

A KMS policy with explicit Deny, allowed Glue via kms:ViaService, break-glass MFA carve-out, STS assumed-role ARN handling, and an Organizations SCP backstop.

Why it works

IAM AdministratorAccess cannot override explicit Deny. A workload-account admin also cannot rewrite the key policy because the Org SCP blocks tampering.

  • OKGlue decrypt is limited to service-context use, including S3/KMS read path support.
  • OKBreak-glass access requires MFA and is designed to be audited.
  • OKThe write-up explains root recovery, SCP enforcement, CloudTrail/EventBridge, and the kill-chain controls.

03Scale

Glue data skew optimization

They asked for

A one-page strategy for handling 10 million GPS pings for a single intersection without increasing Glue Worker Type or Max Capacity.

What I built

A technical note and PySpark skeleton using AQE, two-stage salted aggregation, correct sum/count centroid reconstruction, and time/region S3 partitioning.

Why it works

The hot intersection is split across many shuffle tasks before aggregation. Output partitioning avoids recreating the hot-key problem at write time.

  • OKSalting spreads the hot key across 64 buckets rather than sending all rows to one executor.
  • OKAverage math is handled with weighted sum/count, not biased avg(avg).
  • OKThe note explicitly rejects bigger workers and memory-only fixes.

04DevSecOps

Secure Weather API key rotation

They asked for

A Python Lambda snippet for automated Secrets Manager rotation that verifies the new key against the Weather API before deprecating the old key.

What I built

A full four-step rotation Lambda with createSecret, setSecret, testSecret, and finishSecret. Error responses are redacted to avoid leaking keys.

Why it works

AWSCURRENT only moves in finishSecret after testSecret has successfully called the live API with AWSPENDING. Failure leaves the old key live.

1createSecretIssue new key and store AWSPENDING.
2setSecretNo traffic moves. Old key still serves.
3testSecretCall Weather API with new key.
4finishSecretPromote new key only after test passes.
