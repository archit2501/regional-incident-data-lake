P1Orchestration

Automated ingestion pipeline

They asked for

A serverless S3 zip workflow using Step Functions, MD5 validation, quarantine/SNS routing, extraction, Glue anonymization, and Parquet conversion.

What I built

An ASL state machine with checksum validation, Choice routing, parallel quarantine operations, bounded alerts, retries with jitter, ExtractZip, and Glue startJobRun.sync.

Why it works

Poisoned files never reach analytics. Valid files are anonymized before curated storage. Operational failures alert with filterable metadata.

  • OKMD5 is streamed in Lambda and compared against a manifest field validated as 32-char hex.
  • OKFailed files are copied to quarantine with tags and metadata, then alert SNS.
  • OKPassed files proceed to extraction and Glue anonymization with GPS precision set to 3 decimals.
P2Zero Trust

Ghost Admin defense

They asked for

A KMS key policy that lets Glue decrypt while explicitly denying everyone else, including AdministratorAccess, unless a security condition is met.

What I built

A KMS policy with explicit Deny, Glue via kms:ViaService, break-glass MFA, assumed-role ARN handling, and an Organizations SCP backstop.

Why it works

AdministratorAccess cannot beat explicit Deny. A workload-account admin also cannot rewrite the key policy because the Org SCP blocks tampering.

  • OKGlue decrypt is limited to service-context use, including S3/KMS read-path support.
  • OKBreak-glass access requires recent MFA and is auditable.
  • OKThe explanation covers SCP enforcement, root recovery, CloudTrail/EventBridge, and kill-chain controls.
P3Scale

Glue data skew optimization

They asked for

A one-page strategy for handling 10 million GPS pings for a single intersection without increasing Glue Worker Type or Max Capacity.

What I built

A technical note and PySpark skeleton using AQE, two-stage salted aggregation, correct weighted centroid math, and time/region S3 partitioning.

Why it works

The hot intersection is split across many shuffle tasks before aggregation. Output partitioning avoids recreating the hot-key problem at write time.

  • OKSalting spreads the hot key across 64 buckets instead of one executor.
  • OKAverage math uses weighted sum/count, not biased avg(avg).
  • OKThe note explicitly rejects bigger workers and memory-only fixes.
P4DevSecOps

Secure Weather API key rotation

They asked for

A Python Lambda snippet for automated Secrets Manager rotation that verifies the new key against the Weather API before deprecating the old key.

What I built

A complete four-step rotation Lambda with createSecret, setSecret, testSecret, and finishSecret. Error bodies are redacted to avoid leaking keys.

Why it works

AWSCURRENT only moves in finishSecret after testSecret successfully calls the API with AWSPENDING. Failure leaves the old key live.

1createSecretIssue new key and store AWSPENDING.
2setSecretNo traffic moves. Old key still serves.
3testSecretCall Weather API with the new key.
4finishSecretPromote only after test passes.

GitHub Pages deployment

The repository is a static site. GitHub Pages can serve it directly from the root of the main branch.

index.html at root

How to deploy

Push the repository to GitHub, then set Settings -> Pages -> Deploy from a branch -> main / root. GitHub Pages serves this `index.html` as the homepage.

How to run locally

Run `python3 -m http.server 8088 --bind 127.0.0.1`, then open `http://127.0.0.1:8088/`. The current local server is already serving this site.
